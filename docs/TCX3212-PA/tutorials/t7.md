### Part 0: Understanding Time Series

Before analyzing the full Amtrak dataset, let's start with a small example.

**(a)** A Python list has no dates. We need a `DatetimeIndex` to make it a time series.

- Data values: `[112, 118, 132, 129, 121, 135, 148, 148, 136, 119]`
- Start date: `01-01-2022`
- Frequency: Month-End (`ME`)

Use `pd.date_range()` to create dates, combine the data into a `pd.Series`, and display the time-series object.

**(b)** Extract data from March 2022 to October 2022. Plot the values with dates on the x-axis.

---

**1.** In this exercise, you will explore and analyze monthly ridership data for **Amtrak** (January 1991 – March 2004). Use the `Amtrak.csv` dataset.

| Variable | Description |
|---|---|
| Month | Month of observation in DD/MM/YYYY format |
| Ridership | Number of passengers (in thousands) |

**(c)** Load the Amtrak dataset. Construct a time-series object for Amtrak ridership from 01-01-1991 to 01-03-2004. Plot the monthly ridership with clearly labeled axes. What time series components appear from the plot?

**(d)** Apply a 3rd-order and 12th-order centered moving averages, and plot the smoothed series overlaid on the original. Compare the results and explain how the window sizes affect smoothing.

**(e)** Compute a 12th-order trailing moving average (`center=False`). Explain why a trailing moving average is suitable for forecasting.

**(f)** Using the last value of the 12-month trailing MA, create a naive forecast for the next 6 months. Present a table showing original ridership, trailing MA, and forecasted values. Overlay the 12-month trailing MA on the plot from part (c).

**(g)** Partition the data using the last 36 months as a validation set and the remaining observations as a training set (`nValid = 36`).

**(h)** The **naive forecast** is the simplest forecasting method — it assumes the next value equals the last observed value in the training data. Using the training time series, generate a naive forecast for the entire validation period.

**(i)** The **seasonal naive forecast** accounts for seasonality by repeating the same seasonal pattern from the most recent season. Assuming a 12-month seasonal cycle, generate a seasonal naive forecast for the validation period by repeating the last 12 months of training data.

**(j)** Using the training and validation splits, plot the actual ridership together with the naive and seasonal naive forecasts on a single time-series chart.

**(k)** Implement a Python function `regression_stats(actual, forecast)` that computes **RMSE**, **MAE**, **MPE**, and **MAPE**. Use this function to assess and compare the different forecasting methods.
