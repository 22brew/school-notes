T6: Tutorial 6

Related Lecture: L7: Logistic Regression

Learning Objectives​

  • Perform logistic regression and evaluate its performance using held-out (test) data in Python

Questions​

1. In this exercise, we will use the Smarket (Smarket.csv) dataset, which records percentage returns for the S&P 500 stock index over 1,250 trading days from 2001 to 2005.

VariableDescription
YearYear of observation
Lag1–Lag5Percentage return for previous 1–5 days
VolumeVolume of shares traded (billions)
TodayPercentage return for today
Direction'Down' or 'Up' — whether market had positive/negative return

(a) Load Smarket.csv, display the first few rows, and identify the variable names. Run Smarket.plot(y='Volume'), plot the Volume variable over time, and describe the trend in average daily shares traded from 2001 to 2005.

(b) To model a binary outcome for logistic regression, create a numeric response variable Direction_response by mapping 'Up' to 1 and 'Down' to 0.

(c) Fit a logistic regression model to predict market Direction using Lag1 through Lag5 and Volume. Using statsmodels.formula.api, fit a logistic regression model with Direction_response as the response variable.

Note: Pass family=sm.families.Binomial() to tell statsmodels to run logistic regression.

Use summary() to print the results. Comment on the output.

(d) Determine whether any of the predictors (Lag1 through Lag5 and Volume) are significantly associated with Direction_response.

(e) Interpret the coefficient of Lag1. What does a coefficient of -0.0556 mean in terms of log-odds?

(f) Calculate the odds ratio associated with Volume. Interpret what this implies about the relationship between daily trading volume and the likelihood that the market goes Up.

(g) Predict the probability that the market goes Up for each observation. Convert these probabilities into predicted class labels (0 for 'Down', 1 for 'Up') using a threshold of 0.5. Save as y_pred_class.

(h) Import from sklearn import metrics and compute the confusion matrix using Direction_response and y_pred_class.

(i) Compute the metrics: accuracy, sensitivity, specificity to evaluate performance.

2. Held-out Test Evaluation

The logistic regression model appears slightly better than random guessing, but this is misleading because it was trained and tested on the same 1,250 observations. To properly evaluate the model, we must train it on one portion and test it on another.

(a) Create a Boolean vector train that selects observations from 2001–2004 (rows before 2005 = True). Use loc to extract the training data. Use ~train to select the 252 held-out observations from 2005.

(b) Using only the training data, fit a logistic regression model to predict whether the stock market will go up.

(c) Calculate the predicted probabilities that the stock market will go up for each day in the test data (2005). The model is trained only on pre-2005 observations.

(d) Threshold the fitted probabilities at 0.5 to generate predicted classes for the held-out test data. Assign 1 ("Up") if predicted probability ≥ 0.5 and 0 ("Down") if < 0.5.

(e) Extract the Direction_response column from Smarket and create the training and held-out test labels.

(f) Compute the confusion matrix for predicted vs. actual labels in the held-out test set. Compute accuracy, precision, and specificity. Comment on the model's performance.

(g) Refit the logistic regression model using only Lag1 and Lag2, which showed the highest predictive power. How does removing the other predictors affect performance on the 2005 test data? Compare to the original model.

(h) Given Lag1 = -0.5, Lag2 = 0.3, calculate the predicted probability that Direction_response = Up.
