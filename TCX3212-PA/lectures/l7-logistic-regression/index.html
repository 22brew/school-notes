L7: Logistic Regression

Introduction to Classification​

Classification is a supervised learning task where the response variable YYY takes values in a finite, unordered set (categories), rather than a continuous range.

Unlike regression (which predicts a quantity), classification predicts a class label.

Example: Given a customer's credit card balance and income, predict whether they will default (Yes/No).

Why Not Use Linear Regression for Classification?​

Linear regression can produce predictions outside [0,1][0, 1][0,1], making it unsuitable as a probability estimate for binary outcomes. A value of y^=1.3\hat{y} = 1.3y^​=1.3 or y^=−0.2\hat{y} = -0.2y^​=−0.2 is meaningless as a probability. Threshold heuristics (e.g., predict Yes if y^>0.5\hat{y} > 0.5y^​>0.5) are brittle and can require arbitrary threshold adjustments.

The solution: Logistic Regression, which models the probability of class membership.

Logistic Regression​

The Logistic Function​

Instead of modelling YYY directly, logistic regression models P(Y=1∣X)=p(X)P(Y = 1 \mid X) = p(X)P(Y=1∣X)=p(X) — the probability that the outcome belongs to class 1.

The logistic (sigmoid) function maps any real-valued input to [0,1][0, 1][0,1]:

p(X)=eβ0+β1X1+eβ0+β1X=11+e−(β0+β1X)p(X) = \frac{e^{\beta_0 + \beta_1 X}}{1 + e^{\beta_0 + \beta_1 X}} = \frac{1}{1 + e^{-(\beta_0 + \beta_1 X)}}p(X)=1+eβ0​+β1​Xeβ0​+β1​X​=1+e−(β0​+β1​X)1​

This S-shaped curve never goes below 0 or above 1, making it a valid probability model.

Log-Odds (Logit) Transformation​

To understand the model structure, rearrange:

p(X)1−p(X)=eβ0+β1X(odds)\frac{p(X)}{1 - p(X)} = e^{\beta_0 + \beta_1 X} \qquad \text{(odds)}1−p(X)p(X)​=eβ0​+β1​X(odds)

log⁡ ⁣(p(X)1−p(X))=β0+β1X(log-odds / logit)\log\!\left(\frac{p(X)}{1 - p(X)}\right) = \beta_0 + \beta_1 X \qquad \text{(log-odds / logit)}log(1−p(X)p(X)​)=β0​+β1​X(log-odds / logit)

The logit (log-odds) is linear in XXX — this is what makes logistic regression a "linear" model despite its non-linear output.

Making Predictions​

Choose a decision threshold τ\tauτ:

  • Predict Y=1Y = 1Y=1 if p(X)>τp(X) > \taup(X)>τ
  • Common default: τ=0.5\tau = 0.5τ=0.5
  • Lower threshold (e.g., τ=0.1\tau = 0.1τ=0.1) → more conservative (flag more potential defaults)

Interpreting β1\beta_1β1​​

In linear regression: β1\beta_1β1​ is the average change in YYY per unit increase in XXX.

In logistic regression:

  • A one-unit increase in XXX adds β1\beta_1β1​ to the log-odds.
  • The odds are multiplied by eβ1e^{\beta_1}eβ1​.

Odds change factor=eβ1\text{Odds change factor} = e^{\beta_1}Odds change factor=eβ1​

ScenarioInterpretation

eβ1=1e^{\beta_1} = 1eβ1​=1No change — no association

Percent change in odds: ∣eβ1−1∣×100%|e^{\beta_1} - 1| \times 100\%∣eβ1​−1∣×100%

Example — Default Dataset:

β^1=0.0055 (balance predictor)\hat{\beta}_1 = 0.0055 \text{ (balance predictor)}β^​1​=0.0055 (balance predictor)

  • A $1 increase in balance increases the log-odds of default by 0.0055.
  • The odds of default are multiplied by e0.0055≈1.0055e^{0.0055} \approx 1.0055e0.0055≈1.0055 per $1 increase in balance — approximately a 0.55% increase in odds.

Predicted probability for balance = $1,000:

p=e−10.651+0.0055×10001+e−10.651+0.0055×1000<1%p = \frac{e^{-10.651 + 0.0055 \times 1000}}{1 + e^{-10.651 + 0.0055 \times 1000}} < 1\%p=1+e−10.651+0.0055×1000e−10.651+0.0055×1000​<1%

For balance = $2,000: p≈58.6%p \approx 58.6\%p≈58.6% — much higher risk.

Estimating Coefficients: Maximum Likelihood​

Unlike OLS (which minimises RSS), logistic regression uses Maximum Likelihood Estimation (MLE):

For observation iii, the likelihood of the outcome is:

  • p(xi)p(x_i)p(xi​) if yi=1y_i = 1yi​=1
  • 1−p(xi)1 - p(x_i)1−p(xi​) if yi=0y_i = 0yi​=0

The likelihood function:

L(β0,β1)=∏i:yi=1p(xi)⋅∏i:yi=0(1−p(xi))L(\beta_0, \beta_1) = \prod_{i: y_i=1} p(x_i) \cdot \prod_{i: y_i=0} (1 - p(x_i))L(β0​,β1​)=∏i:yi​=1​p(xi​)⋅∏i:yi​=0​(1−p(xi​))

MLE chooses β^0,β^1\hat{\beta}_0, \hat{\beta}_1β^​0​,β^​1​ to maximise this likelihood — finding parameters that make the observed data most probable. In practice, numerical optimisation is used.

Qualitative Predictors in Logistic Regression​

Same approach as linear regression — create dummy variables:

Example: student variable (Yes/No) with reference = Non-student:

p^(student)=eβ^0+β^1⋅1student1+eβ^0+β^1⋅1student\hat{p}(\text{student}) = \frac{e^{\hat{\beta}_0 + \hat{\beta}_1 \cdot \mathbf{1}_{\text{student}}}}{1 + e^{\hat{\beta}_0 + \hat{\beta}_1 \cdot \mathbf{1}_{\text{student}}}}p^​(student)=1+eβ^​0​+β^​1​⋅1student​eβ^​0​+β^​1​⋅1student​​

Interpretation of β^1=0.4049\hat{\beta}_1 = 0.4049β^​1​=0.4049 (student = Yes):

  • The odds of default for students are e0.4049≈1.499e^{0.4049} \approx 1.499e0.4049≈1.499 times the odds for non-students — approximately 49.9% higher odds.

Multiple Logistic Regression​

Generalises to multiple predictors X=(X1,…,Xp)X = (X_1, \ldots, X_p)X=(X1​,…,Xp​):

log⁡ ⁣(p(X)1−p(X))=β0+β1X1+⋯+βpXp\log\!\left(\frac{p(X)}{1 - p(X)}\right) = \beta_0 + \beta_1 X_1 + \cdots + \beta_p X_plog(1−p(X)p(X)​)=β0​+β1​X1​+⋯+βp​Xp​

Default dataset with 3 predictors:

p^(X)=e−10.869+0.0057⋅balance+0.003⋅income−0.6468⋅student1+e−10.869+0.0057⋅balance+0.003⋅income−0.6468⋅student\hat{p}(X) = \frac{e^{-10.869 + 0.0057 \cdot \text{balance} + 0.003 \cdot \text{income} - 0.6468 \cdot \text{student}}}{1 + e^{-10.869 + 0.0057 \cdot \text{balance} + 0.003 \cdot \text{income} - 0.6468 \cdot \text{student}}}p^​(X)=1+e−10.869+0.0057⋅balance+0.003⋅income−0.6468⋅studente−10.869+0.0057⋅balance+0.003⋅income−0.6468⋅student​

Apparent contradiction: In the single-predictor model, student has a positive coefficient (students have higher default probability). In the multiple model, it has a negative coefficient (for a given balance and income, students are less likely to default).

Why? This is confounding:

  • Students tend to have higher balances — which increases default risk.
  • But for the same balance level, students are actually less risky than non-students.
  • The simple model's positive coefficient is driven by the fact that students carry more debt, not that being a student itself increases risk.

Practical implication: A student is riskier than a non-student if you know nothing about their balance. But given the same balance, the student is less risky. Context and conditioning matter.

Precision vs Recall Tradeoff​

  • Precision penalises false positives (predicting default when customer won't).
  • Recall (Sensitivity) penalises false negatives (missing actual defaults).
  • They are inversely related: lowering the threshold increases recall but decreases precision.

Domain-specific considerations:

  • Medical testing for disease: prioritise recall (don't miss sick patients).
  • Criminal sentencing: prioritise precision (don't wrongly convict innocent people).

Rule of thumb: Choose a threshold that keeps both precision and recall acceptably high for your application.

ROC Curve and AUC​

The ROC (Receiver Operating Characteristic) curve plots TPR (Sensitivity) vs FPR (1 − Specificity) across all possible thresholds.

  • Top-left corner = ideal (high TPR, low FPR).
  • Diagonal line = random classifier.
  • AUC (Area Under Curve): Summarises overall performance across all thresholds. AUC = 1 is perfect; AUC = 0.5 is random.

Q1. For a binary classification task with real-valued inputs in (−∞,∞)(-\infty, \infty)(−∞,∞), which function maps inputs to valid probabilities in [0,1][0, 1][0,1]?

  • A) ✓ 1/(1+e−x)1/(1+e^{-x})1/(1+e−x) — the sigmoid/logistic function
  • B) log⁡(x)\log(x)log(x) — unbounded below, undefined for x≤0x \leq 0x≤0
  • C) exe^xex — unbounded above, always positive

Why A: The sigmoid function is the only one listed that maps all real numbers to (0,1)(0, 1)(0,1).

Q2. In logistic regression, what expression gives the odds of outcome Y=1Y = 1Y=1 given input XXX?

  • A) 1/(1+e−(b0+b1X))1/(1+e^{-(b_0+b_1 X)})1/(1+e−(b0​+b1​X)) — this is the probability.
  • B) b0+b1Xb_0 + b_1 Xb0​+b1​X — this is the log-odds.
  • C) ✓ eb0+b1Xe^{b_0 + b_1 X}eb0​+b1​X

Why C: Odds = p/(1−p)p/(1-p)p/(1−p). Starting from the logistic equation: p/(1−p)=eβ0+β1Xp/(1-p) = e^{\beta_0 + \beta_1 X}p/(1−p)=eβ0​+β1​X.

Q3. A logistic regression is fitted: logit(p)=b0+0.0055⋅balance\text{logit}(p) = b_0 + 0.0055 \cdot \text{balance}logit(p)=b0​+0.0055⋅balance. Which is the correct interpretation of b1=0.0055b_1 = 0.0055b1​=0.0055?

  • A) Each additional $1 in balance increases the probability of default by 0.55%.
  • B) ✓ For every additional $1 in credit balance, the log-odds of default increase by 0.0055.
  • C) For every additional $1 in credit balance, the odds of default increase by exactly 0.0055.

Why B: b1b_1b1​ is the coefficient on the log-odds scale. The change in probability is not constant (it depends on current ppp); the change in odds is multiplicative (×e0.0055e^{0.0055}e0.0055), not additive.

Q4. A logistic regression predicts credit default with student as a predictor (reference = non-student). Student[Yes] coefficient b1=0.4049b_1 = 0.4049b1​=0.4049. Which statement is correct?
