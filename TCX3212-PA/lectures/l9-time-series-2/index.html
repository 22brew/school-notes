L9: Time Series II

Exponential Smoothing Models​

Exponential smoothing methods improve on Simple Moving Average (SMA) by giving more weight to recent observations and exponentially less weight to older ones. Unlike SMA (which weights all window observations equally), exponential smoothing uses all past data with geometrically decaying weights.

There are three variants depending on the structure of the data:

MethodHandles TrendHandles SeasonalityParameters
Simple Exponential Smoothing (SES)NoNoα\alphaα
Double (Holt's)YesNoα\alphaα, β\betaβ
Triple (Holt-Winters)YesYesα\alphaα, β\betaβ, γ\gammaγ

Simple Exponential Smoothing (SES)​

Use case: Time series with a changing mean but no trend and no seasonality.

Smoothing Equation​

The smoothed level ata_tat​ at time ttt is:

at=α⋅yt+(1−α)⋅at−1,0≤α≤1a_t = \alpha \cdot y_t + (1 - \alpha) \cdot a_{t-1}, \qquad 0 \leq \alpha \leq 1at​=α⋅yt​+(1−α)⋅at−1​,0≤α≤1

Expanding recursively:

at=α⋅yt+α(1−α)⋅yt−1+α(1−α)2⋅yt−2+⋯a_t = \alpha \cdot y_t + \alpha(1-\alpha) \cdot y_{t-1} + \alpha(1-\alpha)^2 \cdot y_{t-2} + \cdotsat​=α⋅yt​+α(1−α)⋅yt−1​+α(1−α)2⋅yt−2​+⋯

Each past observation gets weight α(1−α)k\alpha(1-\alpha)^kα(1−α)k — exponentially decaying with lag kkk.

  • Large α\alphaα (close to 1): more weight to recent observations → model adapts quickly, noisier.
  • Small α\alphaα (close to 0): more weight to history → smoother, slower to adapt.

Initialisation: a1=y1a_1 = y_1a1​=y1​.

Forecasting​

SES produces a "flat" forecast — all future forecasts equal the last smoothed level:

y^T+h=aTfor all h=1,2,3,…\hat{y}_{T+h} = a_T \qquad \text{for all } h = 1, 2, 3, \ldotsy^​T+h​=aT​for all h=1,2,3,…

One-step-ahead fitted values: y^t+1=at\hat{y}_{t+1} = a_ty^​t+1​=at​.

Note: SES is equivalent to an ARIMA(0,1,1) model (one nonseasonal difference, MA(1) term, no constant).

Prediction Intervals​

A 95% prediction interval for y^T+h\hat{y}_{T+h}y^​T+h​:

y^T+h±1.96⋅s⋅1+(h−1)α2\hat{y}_{T+h} \pm 1.96 \cdot s \cdot \sqrt{1 + (h-1)\alpha^2}y^​T+h​±1.96⋅s⋅1+(h−1)α2​

where sss is the standard error of residuals. Intervals widen for longer horizons.

Python Example​

from statsmodels.tsa.holtwinters import SimpleExpSmoothing
import pandas as pd

oil = pd.read_csv("Oil.csv", index_col='Year', parse_dates=True)
series = oil['Production']

# Fit SES with alpha = 0.83
model = SimpleExpSmoothing(series).fit(smoothing_level=0.83, optimized=False)
forecast = model.forecast(5)    # 5-step ahead forecast
model.fittedvalues              # in-sample predictions

Double Exponential Smoothing (Holt's Method)​

Use case: Time series that exhibits a linear trend but no seasonality.

Equations​

y^t+k=at+bt⋅k\hat{y}_{t+k} = a_t + b_t \cdot ky^​t+k​=at​+bt​⋅k

where ata_tat​ (level) and btb_tbt​ (trend slope) are updated at each step:

at=α⋅yt+(1−α)(at−1+bt−1)a_t = \alpha \cdot y_t + (1 - \alpha)(a_{t-1} + b_{t-1})at​=α⋅yt​+(1−α)(at−1​+bt−1​) bt=β(at−at−1)+(1−β)bt−1b_t = \beta(a_t - a_{t-1}) + (1 - \beta) b_{t-1}bt​=β(at​−at−1​)+(1−β)bt−1​

  • α\alphaα: smoothing constant for the level.
  • β\betaβ: smoothing constant for the trend.
  • Small β\betaβ: trend changes slowly; the slope is relatively stable.
  • Large β\betaβ: trend adapts quickly to recent data.

Initialise: a1=y1a_1 = y_1a1​=y1​, b1=y2−y1b_1 = y_2 - y_1b1​=y2​−y1​.

Forecasting​

y^T+h=aT+h⋅bT\hat{y}_{T+h} = a_T + h \cdot b_Ty^​T+h​=aT​+h⋅bT​

Unlike SES, the forecast function is trending — it projects the current level with the current slope hhh steps forward.

Example (Australian Air Passengers)​

Smoothing parameters: α=0.8321\alpha = 0.8321α=0.8321, β=0.0001\beta = 0.0001β=0.0001 (nearly constant slope).

Given a1=15.57a_1 = 15.57a1​=15.57, b1=2.102b_1 = 2.102b1​=2.102:

y^2=a1+1⋅b1=15.57+2.102=17.67\hat{y}_2 = a_1 + 1 \cdot b_1 = 15.57 + 2.102 = 17.67y^​2​=a1​+1⋅b1​=15.57+2.102=17.67

Then update:

a2=α⋅y2+(1−α)(a1+b1)a_2 = \alpha \cdot y_2 + (1-\alpha)(a_1 + b_1)a2​=α⋅y2​+(1−α)(a1​+b1​) b2=β(a2−a1)+(1−β)b1b_2 = \beta(a_2 - a_1) + (1-\beta) b_1b2​=β(a2​−a1​)+(1−β)b1​

Triple Exponential Smoothing (Holt-Winters)​

Use case: Time series with both trend and seasonality. LLL = number of seasons per year (L=12L = 12L=12 monthly, L=4L = 4L=4 quarterly).

Additive Holt-Winters​

For constant seasonal amplitude (additive seasonality):

y^t+k=at+bt⋅k+ct+k−L\hat{y}_{t+k} = a_t + b_t \cdot k + c_{t+k-L}y^​t+k​=at​+bt​⋅k+ct+k−L​

Update equations:

at=α(yt−ct−L)+(1−α)(at−1+bt−1)(seasonally adjusted level)a_t = \alpha(y_t - c_{t-L}) + (1 - \alpha)(a_{t-1} + b_{t-1}) \qquad \text{(seasonally adjusted level)}at​=α(yt​−ct−L​)+(1−α)(at−1​+bt−1​)(seasonally adjusted level) bt=β(at−at−1)+(1−β)bt−1(trend)b_t = \beta(a_t - a_{t-1}) + (1 - \beta) b_{t-1} \qquad \text{(trend)}bt​=β(at​−at−1​)+(1−β)bt−1​(trend) ct=γ(yt−at)+(1−γ)ct−L(seasonal index)c_t = \gamma(y_t - a_t) + (1 - \gamma) c_{t-L} \qquad \text{(seasonal index)}ct​=γ(yt​−at​)+(1−γ)ct−L​(seasonal index)

  • ct−Lc_{t-L}ct−L​: seasonal factor from the same season one year ago.
  • Seasonal factor ctc_tct​ is the additive deviation from the (de-seasoned) level.

Multiplicative Holt-Winters​

For growing seasonal amplitude (multiplicative seasonality):

y^t+k=(at+bt⋅k)×ct+k−L\hat{y}_{t+k} = (a_t + b_t \cdot k) \times c_{t+k-L}y^​t+k​=(at​+bt​⋅k)×ct+k−L​

at=α(yt/ct−L)+(1−α)(at−1+bt−1)a_t = \alpha(y_t / c_{t-L}) + (1 - \alpha)(a_{t-1} + b_{t-1})at​=α(yt​/ct−L​)+(1−α)(at−1​+bt−1​) bt=β(at−at−1)+(1−β)bt−1b_t = \beta(a_t - a_{t-1}) + (1 - \beta) b_{t-1}bt​=β(at​−at−1​)+(1−β)bt−1​ ct=γ(yt/at)+(1−γ)ct−Lc_t = \gamma(y_t / a_t) + (1 - \gamma) c_{t-L}ct​=γ(yt​/at​)+(1−γ)ct−L​

Here ctc_tct​ is a multiplicative factor (e.g., 1.24 means that season is 24% above the de-seasoned level).

Python (Holt-Winters)​

from statsmodels.tsa.holtwinters import ExponentialSmoothing

# Multiplicative Holt-Winters (monthly data, L=12)
hw_model = ExponentialSmoothing(
    series,
    trend='add',
    seasonal='mul',
    seasonal_periods=12
)
fit = hw_model.fit(
    smoothing_level=0.441,
    smoothing_trend=0.03,
    smoothing_seasonal=0.002,
    optimized=False
)
forecast = fit.forecast(8)

Performance Evaluation Metrics​

The forecast error at time ttt:

et=yt−y^te_t = y_t - \hat{y}_tet​=yt​−y^​t​

Mean Absolute Deviation (MAD) / Mean Absolute Error (MAE)​

MAD=1n∑t=1n∣yt−y^t∣\text{MAD} = \frac{1}{n}\sum_{t=1}^n |y_t - \hat{y}_t|MAD=n1​∑t=1n​∣yt​−y^​t​∣

  • Robust to extreme errors; easy to interpret.
  • Unit: same as the original series.

Mean Squared Error (MSE) and RMSE​

MSE=1n∑t=1n(yt−y^t)2,RMSE=MSE\text{MSE} = \frac{1}{n}\sum_{t=1}^n (y_t - \hat{y}_t)^2, \qquad \text{RMSE} = \sqrt{\text{MSE}}MSE=n1​∑t=1n​(yt​−y^​t​)2,RMSE=MSE​

  • Penalises large errors more heavily (due to squaring).
  • RMSE is on the same scale as the original series.

Mean Absolute Percentage Error (MAPE)​

MAPE=1n∑t=1n∣yt−y^tyt∣×100%\text{MAPE} = \frac{1}{n}\sum_{t=1}^n \left|\frac{y_t - \hat{y}_t}{y_t}\right| \times 100\%MAPE=n1​∑t=1n​​yt​yt​−y^​t​​​×100%

  • Scale-free; useful for comparing across different series.
  • Undefined when yt=0y_t = 0yt​=0; biased when values are small.

Example:

| yty_tyt​ | y^t\hat{y}_ty^​t​ | ete_tet​ | ∣et∣|e_t|∣et​∣ | APEt\text{APE}_tAPEt​ | |-------|------------|-------|---------|----------------| | 25 | 22 | 3 | 3 | 12.0% | | 28 | 30 | −2 | 2 | 7.1% | | 29 | 30 | −1 | 1 | 3.5% |

  • MAD = (3+2+1)/3 = 2
  • MAPE = (12+7.1+3.5)/3 = 7.5%

Time Series Differencing​

Recall: Stationarity​

Most time series models require stationarity. Non-stationary series (with trends, seasonality, or changing variance) must be transformed first.

Common transformations:

  • Differencing — removes trends.
  • Log transformation — stabilises variance.
  • Detrending — explicitly removes linear trend.

First-Order Differencing​

Subtract the previous observation:

yt′=yt−yt−1,t=2,3,…,ny'_t = y_t - y_{t-1}, \qquad t = 2, 3, \ldots, nyt′​=yt​−yt−1​,t=2,3,…,n

The differenced series has n−1n - 1n−1 values. This removes a linear trend — if the original series grows roughly by a constant amount each period, the differenced series will be approximately stationary.

Second-Order Differencing​

Apply differencing twice (if first-order differencing is insufficient):

yt′′=yt′−yt−1′y''_t = y'_t - y'_{t-1}yt′′​=yt′​−yt−1′​

The twice-differenced series has n−2n - 2n−2 values. Removes quadratic trends. In practice, rarely need to go beyond second-order.

Seasonal Differencing​

Subtract the observation from the same season in the previous cycle:

yt′=yt−yt−my'_t = y_t - y_{t-m}yt′​=yt​−yt−m​

where mmm is the seasonal period (12 for monthly, 4 for quarterly). Also called "lag-mmm differencing."

# First-order differencing
series_diff = series.diff()

# Seasonal differencing (monthly data)
series_seasonal_diff = series.diff(12)

Example: Global Temperature Anomalies​

Original series: non-stationary (clear upward trend).

After first-order differencing (yt′=yt−yt−1y'_t = y_t - y_{t-1}yt′​=yt​−yt−1​): approximately stationary — mean and variance are roughly constant over time.

Connection to ARIMA: The "I" in ARIMA(p, d, q) is the order of differencing needed to achieve stationarity. Examining the ACF of the differenced series helps identify the MA order qqq; the PACF helps identify the AR order ppp.
