Next, we will fit a logistic regression model in order to predict Direction using Lag1 through Lag5 and Volume. The smf.glm function fits generalized linear models, a class of models that includes logistic regression. The syntax of smf.glm is similar to that of smf.lm(), except that we must pass in the argument family=sm.families.Binomial() in order to tell statsmodels to run a logistic regression rather than some other type of generalized linear model.

e) Interpret the coefficient of Lag1 in the logistic regression model. What does a coefficient of -0.0556 mean in terms of log-odds?

Coefficient (β₁) = -0.0731 This means for each 1-unit increase in Lag1, the log-odds of the market going Up decrease by 0.0731.

In simpler terms: if yesterday’s return (Lag1) increases, the model slightly predicts a lower probability of an Up day today.

  1. Converting to odds ratio =e−0.0731≈0.930 Interpretation: For every 1-unit increase in Lag1, the odds of the market going Up decrease by about 7% (since 1−0.930=0.07=1−0.930=0.07).

  2. Significance z = -1.457, p-value = 0.145 Because the p-value > 0.05, this coefficient is not statistically significant at the 5% level. Interpretation: We cannot confidently say that Lag1 has a real effect on predicting an Up day; the observed effect could be due to random chance.

Held out data¶

The logistic regression model appears slightly better than random guessing, but this is misleading because it was both trained and tested on the same 1,250 observations. This training error is overly optimistic and underestimates the true test error. To properly evaluate the model, we must train it on one portion of the data and test it on another.

a) To do this, create a Boolean vector, train that selects observations from 2001–2004 and mark rows before 2005 as ‘True’. Use this vector with the loc method to extract the training data. Then use ~train to select the held-out test data containing the 252 observations from 2005.
